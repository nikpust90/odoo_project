# Odoo 17 — ЖКХ Учёт (замена 1С)

Отдельный Docker-проект с Odoo 17.0 и кастомным модулем `jkh_services`.

---

## Структура проекта

```
odoo_project/
├── docker-compose.yml          # Docker Compose для Odoo + PostgreSQL
├── config/
│   └── odoo.conf               # Конфигурация Odoo
├── extra-addons/
│   └── jkh_services/           # Кастомный модуль ЖКХ
│       ├── __manifest__.py
│       ├── models/
│       │   ├── service_act.py          # Реализация услуг (аналог 1С)
│       │   ├── reconciliation_act.py   # Акт сверки взаиморасчётов
│       │   ├── bank_statement.py       # Банковская выписка
│       │   └── res_partner.py          # Расширение контрагента (ИНН, КПП и т.д.)
│       ├── wizards/
│       │   ├── send_act_wizard.py      # Отправка актов по email
│       │   ├── sbis_export_wizard.py   # Экспорт УПД в XML для СБИС
│       │   └── bank_import_wizard.py   # Импорт банковских выписок (1С формат)
│       ├── views/                      # XML-интерфейсы
│       ├── reports/                    # PDF-шаблоны актов
│       ├── data/                       # Последовательности нумерации
│       └── security/
│           └── ir.model.access.csv
└── docs/
```

## Быстрый старт

### 1. Запуск

```bash
cd odoo_project
docker-compose up -d
```

Odoo будет доступен на `http://ваш-сервер:8069`

### 2. Первоначальная настройка

1. Откройте браузер → `http://localhost:8069`
2. Создайте базу данных `odoo_jkh`
3. Установите модуль **«ЖКХ: Реализация услуг и учёт»**:
   - Настройки → Технические → Обновить список приложений
   - Приложения → найти «ЖКХ» → Установить

### 3. Настройка почты (для отправки актов)

Настройки → Технические → Исходящая почта:
- SMTP сервер (например, `smtp.yandex.ru`, порт `465`, SSL)
- Или скройте в `config/odoo.conf`:
  ```
  smtp_server = smtp.yandex.ru
  smtp_port = 465
  smtp_ssl = ssl
  smtp_user = your@yandex.ru
  smtp_password = yourpassword
  ```

## Функциональность модуля

### Реализация услуг (меню: ЖКХ Учёт → Реализация → Документы реализации)

Аналог документа «Реализация товаров и услуг» из 1С:
- Контрагент, договор, период услуг
- Табличная часть: услуга, количество, цена, НДС
- Статусы: Черновик → Подтверждён → Отправлен → Закрыт
- Создание счёт-фактуры в бухгалтерии одной кнопкой
- Печать PDF-акта выполненных работ
- Отправка PDF по email клиенту
- Экспорт в СБИС (XML формат УПД)

### Акты сверки (меню: ЖКХ Учёт → Реализация → Акты сверки)

- Автозаполнение из проводок бухгалтерии Odoo
- Двусторонняя форма (наши данные + данные контрагента)
- Статусы: Черновик → Отправлен → Подтверждён → Подписан
- Печать PDF в стандартной форме
- Отправка по email контрагенту

### Банковские выписки (меню: ЖКХ Учёт → Банк → Банковские выписки)

- Импорт из формата 1С v8 (`*.txt`, `*.1c`)
- Импорт из CSV (разделитель `;`)
- Автоматическое сопоставление контрагентов по ИНН
- Предпросмотр перед импортом

### Экспорт в СБИС (XML / УПД)

Кнопка **«Экспорт в СБИС»** на документе реализации:
- Формат: УПД (Универсальный передаточный документ)
- Функция: СЧФ / ДОП / СЧФДОП
- Соответствует приказу ФНС ММВ-7-15/820@ (версия 5.01)
- При выборе нескольких документов — формируется ZIP-архив

### Справочники (из стандартного Odoo)

- **Контрагенты** — расширены полями ИНН, КПП, ОГРН, ОКВЭД
- **Виды услуг** — стандартный справочник товаров/услуг Odoo
- **Банковские счета** — в карточке контрагента
- **Налоги** — настройка НДС (Бухгалтерия → Конфигурация → Налоги)
- **Журналы** — банк, касса (Бухгалтерия → Конфигурация → Журналы)

## Совместная работа с rest_api проектом

Odoo работает на отдельных портах (8069, 8072) и не пересекается с вашим текущим REST API проектом. Оба проекта имеют изолированные сети Docker.

---

## Порты

| Сервис | Порт | Назначение |
|--------|------|------------|
| Odoo Web | 8069 | Основной веб-интерфейс |
| Odoo Longpoll | 8072 | Чаты, live-уведомления |
| PostgreSQL Odoo | не открыт внешне | Только внутри Docker-сети |
